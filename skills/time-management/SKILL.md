# 睡眠与时间管理模块 (Time Management Skill)

本模块集成了 aTimeLogger 时间记录和华为运动健康睡眠数据，负责自动生成深度睡眠与时间分析报告。

## 核心计算公式与复核逻辑

### 1. 睡眠周期数
```
睡眠周期 = 睡眠总时长(分钟) / 90
```

### 2. 清醒时长
清醒时长是指睡眠过程中清醒状态的总时长（不包括入睡前和起床后的清醒时间）。程序会在写入数据库前严格按此公式强制复核 AI 提取的数据。
```
清醒时长 = (华为健康醒来时间 - 华为健康入睡时间) - 夜间睡眠总时长
```

### 3. 入睡用时 (关联 aTimeLogger 数据)
```
入睡用时 = 华为健康入睡时间 - aTimeLogger「睡觉/上床」开始时间
```
*(注意：aTimeLogger 返回的时间为 UTC 时间，计算时需 +8 小时转为北京时间)*

### 4. 起床用时 (关联 aTimeLogger 数据)
```
起床用时 = aTimeLogger「睡觉/起床」结束时间 - 华为健康醒来时间
```

## 评价标准

### 入睡用时
| 范围 | 评价 |
|------|------|
| ≤20分钟 | 正常 |
| 21-40分钟 | 较长 |
| >40分钟 | 过长 |

### 起床用时
| 范围 | 评价 |
|------|------|
| ≤10分钟 | 正常 |
| 11-20分钟 | 较长 |
| >20分钟 | 过长 |

### 睡眠评分
| 范围 | 评价 |
|------|------|
| ≥85 | 优秀 |
| 70-84 | 良好 |
| 60-69 | 一般 |
| <60 | 较差 |

## 自动化执行流 (Automated Workflow)

为确保数据百分百准确且避免重复劳动，系统遵循以下严格的执行顺序：

0. **状态检查 (Pre-check)**：在执行“完整分析”前，先行检索数据库。
   - 若当日不存在睡眠数据，或数据**不完整**（缺失核心原始指标）：执行完整流程（从 Step 1 开始），通过 OCR 补全数据。
   - 若当日已存在完整睡眠数据：跳过第 1 步，直接从第 2 步开始复盘。
1. **OCR 原始提取**：由 Vision LLM 从截图中提取原始指标（评分、时长、起止时间等），暂存于内存 Map。
2. **时间数据同步**：系统提取对应日期的 aTimeLogger 记录并先行落库，作为计算基准。
3. **二次指标计算**：利用内存 Map 原始数据（或数据库已有数据）与 aTimeLogger 记录，计算 `sleep_cycles`, `awake_min`, `fall_asleep_min`, `wake_up_min`。
4. **Map 补全与落库**：将计算结果填回内存 Map，确保数据完整后，一次性写入华为睡眠数据库。
5. **深度复盘分析**：将包含计算指标的完整数据包与时间记录发送给文本大模型。

## 数据流转结构示例

在完成 AI 提取与系统复核后，存入数据库的 JSON 数据结构如下：

```json
{
  "date": "2026-05-12",
  "sleep_start": "02:04",
  "sleep_end": "07:41",
  "total_sleep_min": 312.0,
  "deep_sleep_min": 68.0,
  "awake_min": 25.0,
  "awake_count": 4,
  "sleep_score": 75,
  "fall_asleep_min": 82,
  "wake_up_min": 43,
  "sleep_cycles": 3.47,
  "analysis": {
    "summary": "深睡比例偏低，建议调整作息",

    "issues": ["入睡过晚"]
  }
}
```

## 大模型提取 Prompt (Prompt Template)

以下是发送给 Vision LLM 的标准提示词，用于确保数据提取的格式一致性。请严格按照此提示词更新模型行为：

```vision_prompt
【身份设定】
你是一个极致精准的 OCR 数据结构化专家。你的任务是从睡眠截图中提取核心指标。

【核心任务】
请识别图中数据并输出一个纯 JSON 对象，包含以下字段：
1. sleep_date: 格式 YYYY-MM-DD。
2. sleep_score: 睡眠得分 (整数)。
3. deep_sleep_min: 深睡时长 (分钟)。
4. light_sleep_min: 浅睡时长 (分钟)。
5. rem_sleep_min: 快速眼动时长 (分钟)。
6. awake_count: 清醒次数 (数字)。
7. deep_sleep_ratio: 深睡比例 (百分比数字)。
8. light_sleep_ratio: 浅睡比例 (百分比数字)。
9. rem_sleep_ratio: 快速眼动比例 (百分比数字)。
10. sleep_continuity: 睡眠连续性得分 (数字)。
11. breathing_score: 呼吸质量得分 (数字)。
12. analysis_report: 官方解读建议 (字符串)。
13. sleep_start: 入睡时间 (HH:mm)。
14. sleep_end: 醒来时间 (HH:mm)。
15. total_sleep_min: 总睡眠时长 (夜间睡眠，分钟)。

注意：以下指标由系统自动计算，AI 无需提取：sleep_cycles, awake_min, fall_asleep_min, wake_up_min。

【输出样例 - 请严格模仿此 JSON 结构，只改数值，不改结构】
{
  "sleep_date": "2026-05-12",
  "sleep_score": 71,
  "deep_sleep_min": 94,
  "light_sleep_min": 153,
  "rem_sleep_min": 65,
  "awake_count": 3,
  "deep_sleep_ratio": 28,
  "sleep_continuity": 70,
  "breathing_score": 98,
  "analysis_report": "睡眠质量一般，建议增加运动。",
  "sleep_start": "02:04",
  "sleep_end": "07:41",
  "total_sleep_min": 312
}

【强制约束 - 违反将导致解析失败】
1. 禁止输出任何解释性文字、开场白、结尾废话或 Markdown 代码块标识符。
2. 整个输出必须仅包含一个以 { 开头、以 } 结尾的 JSON 对象。
3. 严禁出现 "这里是解析结果"、"好的" 等多余干扰词。
4. 所有时长必须统一换算为“分钟”（例如：1小时20分 ➔ 80）。
5. 缺失数据统一填 null。

【针对高阶模型 (如 Qwen2-VL-72B) 的特别指令】
- 请执行“深度扫描”模式：不要只看文字，要结合卡片的图标、颜色和布局进行二次校验。
- 严禁习惯性填 0：深睡比例、睡眠连续性、呼吸质量通常位于详情页的中下部卡片中，请务必滚动式寻找。
- 核心定位：total_sleep_min 必须提取图中“夜间睡眠”四个字后面紧跟的时长（例如：夜间睡眠 7 小时 20 分 ➔ 440）。
- 日期核对：必须识别截图顶部的具体日期（如 5月11日），并结合图片年份（通常是 2026）输出为标准的 sleep_date (YYYY-MM-DD)。
- 严禁混淆：图中大字标题或最上方的“在床时间”**不是** total_sleep_min，请务必寻找“夜间睡眠”这一行。
```

## 全天复盘分析 Prompt (Full Analysis Prompt)

以下是用于生成深度复盘报告的提示词，供文本大模型使用：

```analysis_prompt
【身份设定】
你是一个专业的时间 management 教练和睡眠专家。你将分析用户一天的睡眠数据和时间记录，提供精准的洞察和行动建议。

【输入数据】
用户将提供一段 **经过程序二次计算补全** 的完整 JSON 数据，包含 OCR 原始指标（评分、各阶段时长）以及系统自动计算出的核心指标（睡眠周期、入睡/起床用时、清醒时长）和 aTimeLogger 活动统计。

【分析任务】
1. **睡眠评估**：评价睡眠质量、时长、连续性以及入睡/起床效率。
2. **效率评估**：分析生产时间占比、娱乐/松鼠病（分心）是否超标。
3. **逻辑闭环**：寻找睡眠状态与次日效率之间的潜在因果。
4. **行动建议**：提供 3-5 条具体、可执行的建议。

【输出要求】
直接输出一个纯 JSON 对象，严禁任何废话，包含以下字段：
- "efficiency_score": 0-100 的综合评分。
- "summary": 一句话总结今日状态（需包含对时间利用率的定性评价）。
- "insights": 字符串数组，包含 3-5 条深度洞察（必须结合 aTimeLogger 中的具体活动名称和时长进行点评，严禁只谈睡眠）。
- "recommendations": 字符串数组，包含 3-5 条行动建议。
- "issues": 数组，列出识别出的主要问题点。
- "time_management_comment": 针对 Part 2 时间管理部分的专项点评。

【风格指南】
语气客观、专业且带有鼓励。避免空洞的套话。
```
